## Git, GitHub, VS Code & Google Colab for Data Science

**Duration:** ~1 hour 50 minutes (110 minutes)  
**Setting:** Classroom with hands-on practice  
**Platforms:** Mac or Windows  
**Goal:** Build confidence using Git and GitHub to save and share your work — no coding experience needed

---

## LEARNING OBJECTIVES

By the end of this 110-minute session, learners will be able to:

1. **Explain what Git and GitHub are** – Understand why version control matters and how Git and GitHub work together, using everyday analogies.

2. **Describe local vs. remote** – Know the difference between files on your own computer ("local") and files stored on GitHub in the cloud ("remote").

3. **Use the essential Git commands** – Stage, commit, push, and pull changes with confidence, either through VS Code or the terminal.

4. **Connect VS Code to GitHub and stay in sync** – Clone a repository, make changes in VS Code, and push updates to GitHub using the built-in Source Control panel.

5. **Understand what Google Colab is** *(Optional)* – Know what Colab is, why it's useful for data science, and how it connects to GitHub.

## QUICK SUMMARY FOR LEARNERS

**Git** is like a "save history" system for your work — it takes snapshots so you can always go back.

**GitHub** is where those snapshots are stored in the cloud, so you can access your work from anywhere and share it with others.

**VS Code** is the tool you'll use to write and edit files, with a built-in panel that makes Git easy to use without memorising commands.

**The core workflow:**
1. Make a change in VS Code
2. Stage it (select what to save)
3. Commit it (take the snapshot)
4. Push it (send to GitHub)
